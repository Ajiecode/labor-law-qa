# Dify 劳动法问答 - 完整创建步骤

## 步骤1：创建知识库

1. 登录 Dify 后，在左侧菜单点击 **「知识库」**
2. 点击右上角 **「创建知识库」**
3. 填写：
   - 名称：`劳动合同法`
   - 描述：劳动法相关问答知识库
4. 点击 **「创建」**

5. **上传文档**：
   - 点击「添加文档」
   - 选择上传 `labor_law.txt` 文件
   - 点击「下一步」

6. **分段设置**：
   - 分段方式：通用
   - 点击「保存并处理」

7. 等待处理完成（页面显示"已就绪"）

---

## 步骤2：创建工作流应用

1. 点击左侧菜单 **「工作室」**
2. 点击 **「创建应用」**
3. 选择 **「工作流」** → **「创建空白工作流」**
4. 名称：`劳动法问答`
5. 点击 **「创建」**

---

## 步骤3：配置开始节点

进入工作流编辑页面后：

1. 点击画布上的 **「开始」** 圆形节点
2. 右侧弹出设置面板
3. 点击 **「+ 添加变量」**
4. 设置：
   ```
   变量名称: query
   类型: 文本
   显示名称: 用户问题
   必填: ✅ 开启
   ```
5. 点击「保存」

---

## 步骤4：添加知识检索节点

1. 在「开始」节点下方或右侧，找到 **「+」** 按钮（连接点）
2. 点击 **「+」** → 弹出节点列表
3. 选择 **「知识检索」**
4. 点击新添加的「知识检索」节点
5. 右侧设置：
   - 点击 **「选择知识库」**
   - 勾选「劳动合同法」
   - 设置 Top K: 5
6. 点击「保存」

---

## 步骤5：添加LLM节点

1. 在「知识检索」节点后面点击 **「+」**
2. 选择 **「LLM」**
3. 右侧设置：
   
   **选择模型：**
   - 点击模型选择框
   - 选择你配置好的模型（如 GPT-4、DeepSeek 等）
   
   **系统提示词：**
   - 找到「系统提示词」输入框
   - 粘贴以下内容：

```
你是一个专业的劳动法律咨询助手，专门帮助用户解答关于《中华人民共和国劳动合同法》的问题。

回答要求：
1. 只基于提供的知识库内容回答，不要编造法律条文
2. 回答要准确、清晰、简洁
3. 如果问题与劳动合同法无关，礼貌地说明无法回答
4. 引用相关法条时，说明具体条款编号
5. 适当使用表格或列表使回答更清晰
6. 对于复杂问题，先给出结论，再解释原因
```

4. 找到「上下文」或「上下文变量」：
   - 点击选择变量
   - 选择 `{{knowledge_retrieval.result}}` 或类似选项

5. 找到「用户输入」或「提示词变量」：
   - 输入 `{{start.query}}`

6. 点击「保存」

---

## 步骤6：添加结束节点

1. 在「LLM」节点后面点击 **「+」**
2. 选择 **「结束」**
3. 右侧设置：
   - 点击 **「+ 添加输出变量」**
   - 变量名称: `answer`
   - 变量值: 输入 `{{llm.text}}` 或选择 LLM 的输出
4. 点击「保存」

---

## 步骤7：检查连接

确保画布上节点按顺序连接：

```
[○ 开始] ----> [◇ 知识检索] ----> [□ LLM] ----> [○ 结束]
```

如果没有连接线，点击节点右侧的「●」拖拽到下一个节点的左侧「○」

---

## 步骤8：测试

1. 点击右上角 **「运行」** 按钮
2. 在「用户问题」输入框输入：
   ```
   试用期最长多久？
   ```
3. 点击 **「运行」** 按钮
4. 查看右侧输出结果

---

## 步骤9：发布

1. 点击右上角 **「发布」**
2. 点击 **「发布更新」**
3. 点击 **「访问API」**
4. 复制：
   - **App ID**
   - **API Key**

---

## 步骤10：开始使用

1. 用浏览器打开项目中的 `index.html`
2. 在配置区域填入：
   - API URL: `https://api.dify.ai/v1`
   - API Key: 粘贴你的 API Key
   - App ID: 粘贴你的 App ID
3. 开始提问！

---

## 常见问题

**Q: 找不到某个节点？**
A: 点击节点之间的「+」就会弹出节点列表

**Q: 变量找不到？**
A: 点击输入框右侧的「{}」或「变量」按钮选择

**Q: 发布后不能调用？**
A: 确保已经点击「发布更新」，然后获取新的API凭证
